Detailed Description

Sends ExtrapTrackData via ZeroMQ RADIO socket (UDP multicast)

Implements both IAdapter (for lifecycle management) and IExtrapTrackDataOutgoingPort (for domain service integration). Uses background worker thread for non-blocking sends (~20ns enqueue).

Thread Safety:

  • Uses mutex-protected queue for thread-safe message passing
  • Background worker thread handles actual ZMQ transmission
  • Non-blocking sendExtrapTrackData() for real-time performance
Class Invariants:
When running_, socket is connected and ready to send
