Detailed Description

Encapsulates a complete message processing pipeline.

Groups related incoming and outgoing adapters for a specific message type. Each pipeline runs in isolation with its own thread(s).

Design Rationale:

  • Class instead of struct: Has behavior (start/stop/isRunning)
  • Encapsulation: Internal state management
  • Single Responsibility: Only manages one message type's flow
Class Invariants:
name_ is non-empty after construction
Class Invariants:
At least one adapter (incoming or outgoing) should be set
